技术突破

Dual-Trace Memory：跨 Session 记忆准确率 +20pp，零额外 token 成本 4/14

每个存储事实额外携带一条「场景叙事轨迹」（记忆是在什么情境下获得的），而非只存干燥事实。LongMemEval-S 实测：73.7% vs 53.5%（+20.2pp，p<0.0001），零额外 token 成本。提升集中在时序推理（+40pp）、知识更新追踪（+25pp）、多 session 聚合（+30pp）——单 session 检索无增益。

对 ORCHE 意义：当前 memory 系统以 flat 文件为介质，缺乏时序与情境标注。Dual-trace 思路可作为 Layer 1 memory 格式演化的设计参考——写入 memory 条目时附带「发现情境」（触发场景、Kevin 当时说了什么）而非仅存结论。arXiv 2604.12948

Scaling Test-Time Compute for Agentic Coding（RTV + PDR） 4/16 Academic ORCHE 相关
将每次 coding agent rollout 压缩为结构化摘要（假设、进度、失败模式），再做 parallel scaling（Recursive Tournament Voting）+ sequential scaling（Parallel-Distill-Refine）。Claude-4.5-Opus 在 SWE-Bench Verified 从 70.9% 升至 77.6%，Terminal-Bench v2.0 从 46.9% 升至 59.1%。核心洞察：长 horizon agent 的 test-time scaling 本质是「表示、选择与重用」问题，不是更多采样。与 ORCHE session wrap-up 写入 state.md 的机制同构——提供形式化方向。arXiv 2604.16529
MemoSight：CoT KV Cache 压缩 66% + 推理提速 1.56x 4/16 Academic
统一 context compression 与 multi-token prediction 于单一框架，通过特殊 token + position layout 同时压缩 KV cache 和并行生成候选 token。四个 reasoning benchmark 上 KV cache footprint 降 66%，推理速度提升 1.56x，且不低于当前最好的 CoT 压缩方法。若被 Anthropic / vLLM 采纳，可降低 ORCHE 重度 session 的 token 消耗。arXiv 2604.14889
