Anthropic 4/23 Postmortem：3 个 harness bug 导致月级质量下滑

4/23 Anthropic Engineering 官方披露 3/4-4/20 期间三个独立 bug 叠加导致 Claude Code 用户体验下滑：(1) 默认 effort 静默降级至 medium（3/4，4/7 回滚）；(2) idle >1h 后每 turn 清除 thinking 上下文（3/26，4/10 修复）；(3) 减少 verbosity 的 system prompt 与 Opus 4.7 组合拉低代码质量约 3%（4/16，4/20 回滚）。

三 bug 叠加产生用户感知的「遗忘」「回答变短」「配额暴耗」投诉。API 不受影响。Simon Willison 4/24 撰文深度分析，指出长时间闲置 session 受 idle bug 影响最大。

DeepSeek V4：Hybrid Attention 使 1M context KV cache 降至 10%

4/24 DeepSeek 发布 V4-Pro（1.6T 总参/49B 激活）与 V4-Flash（284B/13B 激活），两者默认 1M token context。核心架构突破「Hybrid Attention」结合 CSA（压缩率 1/4）+ HCA（压缩率 1/128），使 1M context 下单 token FLOPs 仅为 V3.2 的 27%、KV cache 仅 10%。

API 已上线（V4-Pro $1.74/$3.48 per M tokens，V4-Flash $0.14 input）；首个针对华为 Ascend 950 优化的 DeepSeek 模型；性能自称对标 Claude Opus 4.6 / GPT-5.4（落后约 1-2 代）。

⚠ 重要 caveat：尽管多家媒体描述为「Apache 2.0 开源」，但官方 GitHub 仓库 deepseek-ai/DeepSeek-V4 当前 404（gh api 已验证），仅 API 可用，开源权重尚未公开释出。需持续观察。
